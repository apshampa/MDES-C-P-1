Module 01

QUIZ BUILDING BOOTCAMP

From absolute scratch. No assumptions.

Step 01.

THE BARE HTML BONES

Every quiz needs a box to live in. We write pure HTML containers to structure the content before touching design.

<div class="quiz-box"> <h1 id="question-text">Question goes here</h1> <div id="options-area"> <button>Option A</button> <button>Option B</button> </div> <button id="next-button">Next</button> </div>

Step 02.

CSS DRESSING

We use CSS flexbox to force everything to stack cleanly and look like an app rather than a Word document.

.quiz-box { display: flex; flex-direction: column; width: 500px; padding: 2rem; background: #111111; border-radius: 12px; } button { padding: 1rem; }

Step 03.

JAVASCRIPT STATE

We cannot write 50 different HTML pages for 50 questions. We write ONE page, and cycle the data using Javascript objects.

const quizData = [ { q: "What is the best font?", responses: ["Inter", "Arial", "Comic Sans"], answerIndex: 0 }, { q: "What color is HEX #000?", responses: ["White", "Black"], answerIndex: 1 } ];

Step 04.

THE DOM BRIDGE

We use Javascript to grab the empty HTML elements and shove our array data inside them.

let currentIndex = 0; let myTextElement = document.getElementById("question-text"); myTextElement.innerText = quizData[currentIndex].q;

Step 05.

MAKING IT CLICK

Without functions, the quiz never moves. We attach an action to the "Next" button.

function goToNext() { currentIndex = currentIndex + 1; if (currentIndex < quizData.length) { myTextElement.innerText = quizData[currentIndex].q; } else { alert("Quiz Finished!"); } }

Module 02.

SCORING STRATEGIES

Every time they click next, we check their answer math before proceeding.

CLASSIC (+1 / 0)
if (correct) score += 1;
COMPETITIVE (+4 / -1)
if (correct) score += 4;
else if (attempted) score -= 1;

Module 03.

ADVANCED QUIZ FORMS

Some quizzes require completely different layouts entirely.

  • True / False: Use `flex-direction: row;` to make binary buttons stack horizontally.
  • Select All That Apply: Use `` instead of `radio` to allow multiple clicks without the browser resetting them.

Module 03.

PERSONALITY PROFILING

Tally names instead of numbers for outcome quizzes (e.g., Which Hogwarts House are you?).

let personaScores = { "Gryffindor": 0, "Slytherin": 0 }; let userSelection = "Gryffindor"; personaScores[userSelection] += 1; let winner = Object.keys(personaScores).reduce((a, b) => personaScores[a] > personaScores[b] ? a : b );

Module 04.

VISUALIZING RESULTS

Show logic, don't just tell. We can use modern CSS math to dynamically draw circular graphics.

let percentage = (score / maxScore) * 100; let circle = document.querySelector('.circular-progress'); circle.style.background = `conic-gradient(#3b82f6 ${percentage * 3.6}deg, #333 0deg)`;

Module 04.

BAR GRAPH REPRESENTATIONS

We can achieve horizontal graphic representations easily by changing a DIV's width percentage.

.bar-track { width: 100%; background: #333; } .bar-fill { background: blue; transition: width 1s; } // In JS: document.getElementById('bar-fill').style.width = percentage + "%";

Module 04.

CONDITIONAL GIFS

We read the final percentage, and load different GIPHY URLs into an image tag.

let imageTag = document.getElementById('result-image'); if (percentage >= 80) { imageTag.src = "https://media.giphy.com/media/xT0xezQGU5xCDJuCPe/giphy.gif"; // Happy } else { imageTag.src = "https://media.giphy.com/media/T1WqKkLY753dZghV8E/giphy.gif"; // Sad }
