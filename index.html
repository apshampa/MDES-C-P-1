Module 01: The Skeleton

HTML & SEMANTICS

HTML (HyperText Markup Language) provides the structure. It tells the browser what things are, not how they look.

"If you turn off all the styling (CSS), a well-structured document should still be perfectly readable and logical."
  • <h1> for main headings
  • <p> for paragraphs
  • <button> for interactive actions

Module 02: The Aesthetic

CSS & THE BOX MODEL

Every single element on a web page is a rectangle. CSS (Cascading Style Sheets) decorates these rectangles.

  • Margin: Space outside the box.
  • Border: The outline of the box.
  • Padding: Space inside the box, pushing content inward.

Module 03: The Behavior

JAVASCRIPT & INTERACTIVITY

JavaScript listens for events (clicks, scrolls) and changes the HTML or CSS in response.

1. Find the element: document.getElementById('btn')
2. Listen to it: .addEventListener('click')
3. Do something: Change color, fetch data, show modal.

Module 04: The Grid

LAYOUTS & GRIDS (MOBILE VS DESKTOP)

We don't draw boxes randomly. We align them to mathematical grids.

  • Columns: The main areas where content lives. Usually 12 on Desktop, 4 on Mobile.
  • Gutters: The empty space between the columns. (Typically 16px - 24px).
  • Margins: The empty space on the far left and right edges of the screen.
  • Responsive: As the screen shrinks, the number of columns decreases, stacked vertically.

Module 04: The Tool

CHROME DEVTOOLS

The ultimate tool for a designer is the browser's inspector. Right-click any website and hit "Inspect Element".

  • Elements Panel: See the real HTML structure.
  • Styles Panel: Live-edit CSS without writing code (perfect for fixing spacing issues in QA).
  • Network Panel: See why an image is taking so long to load.

Module 06: The Vocabulary

UNITS: PX vs REM vs DP

Designers think in points or pixels. Browsers think in scalable units.

  • px (Pixels): Absolute. 16px is always 16px. Bad for accessibility if users change default font sizes.
  • rem (Root EM): Relative. 1rem = the browser's default font size (usually 16px). If a user needs larger text, rems scale automatically.
  • dp / pt: Density-independent pixels (Android/iOS). Used in Figma to represent physical size across different screen densities.
  • Rule: Use rem for typography and spacing on the web to respect user preferences.

Conclusion

THE DESIGNER-DEVELOPER HANDOFF

What Developers Hate

  • "Make it pop more" (Subjective)
  • Unused CSS files in handoff
  • Missing hover/active states in Figma
  • Inconsistent spacing (12px here, 13px there)

What Developers Love

  • Design Tokens (--color-primary)
  • Consistent grids and 4px/8px baselines
  • Interactive prototypes that demonstrate flow
  • Designers who understand the Box Model
